Server & API¶

The alertad server receives alerts from multiple sources, correlates and de-duplicates them, and makes the alerts available via a JSON API.

Alerts can be intercepted as they are received to modify, enhance or reject them using pre-receive hooks. Alerts can also be used to trigger actions in other systems after the alert has been processed using post-receive hooks.

There are several integrations with popular monitoring tools available and webhooks can be used to trivially integrate with Pingdom and AWS Cloudwatch.

Plug-ins¶

Plug-ins are small python scripts that can run either before or after an alert is saved to the database. This is achieved by registering prereceive and post-receive hooks.

Alert Database¶

The document-oriented datastore MongoDB is used as the backend for Alerta to store alerts, heartbeats, API keys and the user exception list. It can be used as a stand-alone server or in a replica set for high availability.

MongoDB Settings¶

# MongoDB
MONGO_HOST = 'localhost'
MONGO_PORT = 27017
MONGO_DATABASE = 'monitoring'

MONGO_USERNAME = 'alerta'
MONGO_PASSWORD = None
